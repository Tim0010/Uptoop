## Tips for Writing Release Notes

### Best Practices
1. **Keep it concise** - Users scan quickly
2. **Use emojis** - Makes it more engaging (but don't overdo it)
3. **Highlight benefits** - Focus on what users gain
4. **Be specific** - Mention actual features
5. **Include contact** - Provide support email
6. **Stay positive** - Use encouraging language
7. **Update regularly** - Keep users informed

### Character Limits
- **Maximum:** 500 characters per language
- **Recommended:** 300-400 characters for readability
- **Minimum:** At least describe key features

### What to Include
- ‚úÖ New features
- ‚úÖ Major improvements
- ‚úÖ Bug fixes (if significant)
- ‚úÖ Call to action
- ‚úÖ Support contact
- ‚ùå Technical jargon
- ‚ùå Negative language
- ‚ùå Excessive detail
